# astrbot_plugin_user_terms

用户条款签署控制插件。插件提供两个手动 SID 列表：

- 用户 SID 列表：命中的用户只在私聊里需要同意条款后才能继续使用。
- 群组 SID 列表：命中的群组需要先同意条款；群组同意后，该群内成员整体放行。
- 条款管理员 SID 列表：命中的用户可以通过聊天指令管理签署状态。
- 禁用用户 / 群组 SID 列表：命中后会直接提示联系开发者。

例如同时配置：

- 群组 SID：`114514`
- 用户 SID：`1919810`

当 `114514` 群组同意条款后，`1919810` 即使还没有在用户作用域签署，也可以在该群里正常和机器人互动；但他私聊机器人时仍然需要按用户作用域单独同意。群聊里只看群组条款，不会因为用户 SID 也在列表中而提示用户条款。

## 使用

1. 将本目录放到 AstrBot 的 `data/plugins/astrbot_plugin_user_terms` 下。
2. 在 AstrBot WebUI 插件配置里手动填写 `用户 SID 列表`、`群组 SID 列表`。
3. 修改 `用户条款正文`、`首次条款提示`、`未阅读时提前同意提示`、`同意后的提示`、`拒绝后的提示` 和 `无效回复提示`。
4. 用户在群里普通聊天不会触发检查；只有艾特机器人、使用唤醒/命令前缀，或命中其它插件的交互 handler 时，插件才会检查条款并要求回复 `我已知晓并同意遵守条款所有内容`。
5. 用户第一次交互就直接发送完整同意句时，不会直接签署；插件会先发送条款和 `未阅读时提前同意提示`，下一次再发送完整同意句才会记录同意状态。
6. 进入插件详情页中的 `签署面板`，可以查看当前签署记录、禁用列表、目标统计，并发布新条款。

## 聊天管理指令

配置在 `条款管理员 SID 列表` 中的用户可以使用：

```text
/terms status [user|group] [目标SID] [用户SID]
/terms set user <用户SID> <accepted|pending|rejected>
/terms set group <群组SID> <accepted|pending|rejected>
/terms reset all
/terms reset user <用户SID>
/terms reset group <群组SID>
/terms publish <新条款正文>
```

被禁用用户会全局生效：如果 `114514` 在禁用用户列表中，他在私聊中会被拒绝，在任何群组中尝试和机器人互动时也会被拒绝。

## 发布新条款

在 `签署面板` 中填写新条款正文并点击 `发布并重置` 后，插件会：

- 创建新的条款版本。
- 将所有已记录签署状态重置为 `pending`。
- 下一次受控用户或群组成员触发机器人时，要求重新同意新条款。

## 数据

SQLite 数据库会保存到：

```text
data/plugin_data/astrbot_plugin_user_terms/terms.sqlite3
```

其中：

- `protected_targets`：记录配置过的用户或群组目标。
- `term_acceptances`：记录用户作用域或群组整体作用域下的签署状态。
- `terms_revisions`：记录每次发布的条款版本。
